Bruncher

Google Maps API, UI, Materialize

In post-quake Christchurch, I often found myself spending far too much of my time driving around the CBD hoping to find a cafe that was open for Brunch. Thatâ€™s why I created Bruncher.

When designing Bruncher it was important that the app gave real-time information on nearby cafes, with open/close information, detailed directions and was easy to use on mobile devices. Using the Google Maps Javascript API in conjunction with the Materialise CSS framework I was able to do all these things.

The app has an easy-to-use interface which allows users to locate local cafes via geolocation or manual input and determine whether they are open or closed. The app also offers the users detailed directions and is responsive across a range of devices.
